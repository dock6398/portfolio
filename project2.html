Problems & Solution

1. GSAP

(1)GSAP 사용 시 마우스 이벤트를 발생 시킬 영역을 조정하는데 어려움이 발생함. -> innerWidth(Height와 screen.width(height)를 이용하여 영역을 조정 하려고 함. (2) 선택한 코드가 잘못된 선택이었는지 여전히 마우스 이벤트 효과가 발생을 하지않음. -> 다른 방법을 찾던 중에 ClientX(Y) - window.innerWidth(height) 값을 사용시 이벤트 발생을 확인하여 문제를 해결함.

2. stroke

(1) stroke를 이용한 애니메이션 효과를 이용할 때 stroke-dasharray 태그와 stroke-dashoffset의 애니메이션 효과로 선의 시작점을 지정해주는 것과 간격 조절을 계속 수정하는 어려움이 발생. -> stroke-dasharray와 stroke-dashoffset를 수정하며 웹 페이지에 나타나는 이벤트 효과를 확인하며 문제를 해결함. (2)선의 시작점과 간격의 조절을 맞추고 애니메이션 효과를 확인 중 home 영역에 적용한 애니메이션이 잘려져 보여지는 문제점이 발생. -> stroke를 넣은 svg 이미지가 고정 사이즈가 아님으로 인해 home 영역을 벗어나 사이즈를 고정으로 주어 해결함.

3. Swiper

(1) Swiper가 적용되지 않고, 슬라이드 버튼이 화면에 보이지 않는 문제가 발생함. -> .swiper_slide 와 .text_wrapper 을 감싸는 영역을 작성하여 이벤트가 발생하지 않는 문제는 해결함. (2) 슬라이드는 정상적으로 작동을 하게 되었지만, 슬라이드에 점들이 보이지 않는 문제는 해결이 되자 못함. -> 슬라이드 버튼의 요소가 보이지않아 ul과 li태그를 div을 이용하여 교체하니 버튼이 나와 문제를 해결함.

4. hover

웹 사이즈 960 ~1240 사이 창업안내 요소의 움직임이 발생하여 양 옆 요소들이 움직이는 문제가 발생하게 됨 -> 문제 원인의 요소의 a에 padding값을 조절하며 ::before의 준 padding의 값도 조정하여 문제 해결

5. svg 사용

FORENA 퍼블리싱 중에는 이미지을 png 파일을 기본적으로 이용을 해서 제작을 함. -> 이번 Dal.komm 퍼블리싱 중에 이미지를 이용 시 png보다 svg를 사용을 하여 웹에 친화적으로 접근을 하여고 함.

6. 반응형 웹

(1) 웹 사이즈의 따라서 요소들의 position의 값이 바뀌는 모습을 확인하여 요소들의 배치에 어려움을 겪음. -> @media를 이용하여 웹 사이즈의 따라서 요소들의 position의 값이 바뀌도록 설정함. (2) 웹 사이즈에 따라서 App 영역에서 앱의 이미지에 양 옆으로 보이게 배치한 ul가 일정 요소가 가려지는 문제가 발생 . -> ul에서 가려지는 부분에 margin-left 값을 웹 사이즈 별로 다르게 주어서 가려지는 요소를 없도록 해결함.

Review

사이트를 보면서 내 방식대로 똑같은 모습을 만들어가는데 다양한 효과들로 처음에는 막막해 보였다. 다양한 이벤트 효과들을 생각하며 요소들의 구조를 작성하고 지우며 반복을 하면서 화면을 보면서 요소들의 구조를 잘 잡아가고 있는지 계속 생각을 하게 되었다.처음 사용하는 GSAP와 익숙하지 않은 FadeIn, Swiper 등을 사용하는데 복잡하고 어려움을 느꼈고, flex를 이용한 요소들의 배치와 toggle을 이용한 이벤트를 어렵지 않게 작성할 수 있었고, 완성된 결과물을 보면서 더 다양한 이벤트들을 공부해야겠다고 생각을 하게 되었다.
