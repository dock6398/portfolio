Overview

커피 브랜드의 달콤 반응형 웹 사이트 퍼블리싱

커피 브랜드의 Dal.komm의 공식 홈페이지를 퍼블리싱 했습니다.
배운 기술들을 토대로 사이트를 내 식대로 똑같이 만들어 보았습니다.

Problems & Solution

1. GSAP

(1)GSAP 사용 시 마우스 이벤트를 발생 시킬 영역을 조정하는데 어려움이 발생함 innerWidth(Height와 screen.width(height)를 이용하여 영역을 조정 할려고 함. (2) 선택한 코드가 잘못된 선택이었는지 여전히 마우스 이벤트 효과가 발생을 하지않음. 다른 여러 방법의 코드를 이용하여 영역을 조정할려고 하면서 찾다가
ClientX(Y) - window.innerWidth(height) 값을 사용시 이벤트 발생을 확인하여 문제를 해결함.

2. stroke

(1) stroke를 이용한 애니메이션 효과를 이용할 때 stroke-dasharray 태그와 stroke-dashoffset의
애니메이션 효과로 선의 시작점을 지정해주는 것과 간격 조절을 계속 수정하는 어려움이 발생. stroke-dasharray와 stroke-dashoffset를 수정하며 체크하여 문제를 해결함. (2)선의 시작점과 간격의 조절을 맞추고 애니메이션 효과를 확인 중 home 영역에 적용한 애니메이션이
잘려져 보여지는 문제점이 발생. stroke를 넣은 svg 이미지가 고정 사이즈가 아님으로 인해 home 영역을 벗어나 사이즈를 고정으로 주어 해결함.

3. Swiper

(1) Swiper가 적용되지 않고, 슬라이드 버튼이 화면에 보이지 않는 문제가 발생함. .swiper_slide 와 .text_wrapper 을 감싸는 영역으로
새로 작성하여 슬라이더가 진행되지 않는 문제는 해결함. (2) 슬라이드는 정상적으로 작동을 하게 되었지만, 슬라이드에 점들이 보이지 않는 문제는 해결이 되자 못함. 슬라이드 버튼의 요소가 보이지않아 ul과 li태그를 div을 이용하여 교체하니 버튼이 나와 문제를 해결함.

4. hover

윈도우 사이즈 960px ~1240px 사이에서 창업안내 영역을 hover 발생시 움직임이 발생함.
움직임이 발생하니 양옆의 다른 요소들도 움직임이 발생하며 영역 밖으로 벗어나는 문제가 발생하게 됨 hover 효과가 발생시 문제가 발생하는 창업안내쪽 a의 padding값을 조정함.
a의 padding값 수정으로 ::before 포지션 값도 같이 수정해 문제 해결

5. svg 사용

저번 FORENA 퍼블리싱에서는 이미지를 사용 시 png 파일을 이용을 함. 이번 Dal.komm 퍼블리싱을 진행하면서 이미지를 이용할 때 png보다 svg를 사용을 하여
좀 더 웹에 친화적으로 접근을 하여고 함.

Review

@약점

처음 사용 또는 익숙하지 않는 GSAP와 마우스 이벤트, FadeIn의 기술력이 부족했다.
나중에 css와 js를 작업할다 보면 요소들의 추가되거나 삭제 되거나 등
수정을 할 부분들이 발생하게 되는것에 요소들의 작성을 할 때 좀 더 잘 다듬어야할 것 같다.

@강점

사용해보지 않은 GSAP나 아직 익숙하지 않는 Swiper, Fadein, 윈도우 영역 등
활용해보면서 경험을 쌓을 수 있었던 것 같다.

@전체적인 총평

사이트를 보면서 내 방식대로 똑같은 모습을 만들어가는데 다양한 효과들로 처음에는 막막해 보였다.
다양한 이벤트 효과들을 생각하며 요소들의 구조를 작성하고 지우며 반복을 하면서
화면을 보면서 요소들의 구조를 잘 잡아가고 있는지 계속 생각을 하게 되었다.
처음 사용하는 이벤트 나 익숙하지 않은 이벤트를 사용하는 것이 정말 복잡하고 어려움을 많이 느꼈지만
완성된 결과물을 보면서 더 다양한 이벤트들을 공부해야겠다고 생각을 하게 되었다.
